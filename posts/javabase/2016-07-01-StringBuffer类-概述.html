StringBuffer类-概述

发表于 2016-07-01   |   分类于 java基础   |     |   阅读次数

StringBuffer

StringBuffer是字符串缓冲区，即一个容器

特点

  • 它的长度是可以变化的
  • 可以直接操作多个数据类型
  • 最终会通过toString方法变成字符串

    功能：CURD

  • C：create 存储
    • StringBuffer append()：将指定数据作为参数添加到已有数据结尾处
    • StringBuffer insert(index,数据)：可以将数据插入到指定index（位置）
  • U：update 修改
    • StringBuffer replace(start,end,String str)
    • void setCharAt(int index,char ch)
  • R：read 获取
    • char charAt(int index)
    • int indexOf(String str)
    • int lastIndexOf(String str)
    • int length()
    • String substring(int start,int end)
  • D：delete 删除
    • StringBuffer delete(start,end)：删除缓冲区中的数据，包含start,不包含end
    • StringBuffer deleteCharAt(index)：删除指定位置的字符
  • 反转
    • String reverse();
  • 将缓冲区中的指定数据存储到指定字符数组中
    • void getChars(int sreBegin,int srcEnd,char[] dst,int dstBegin)

StringBuilder

  • JDK1.5 版本之后出现了StringBuilder
    • StringBuffer是线程同步
    • StringBuilder是线程不同步

      以后开发建议使用StringBuilder

  • 升级版本的三个因素
    • 提高效率
    • 简化书写
    • 提高安全性
